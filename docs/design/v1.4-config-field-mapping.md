# v1.4 ConfigMap Field Mapping: Upstream to Kubernaut CR

**Version**: 1.0
**Upstream baseline**: kubernaut v1.4.0-rc3
**Status**: Authoritative Reference

## Overview

This document maps every configuration field from all 10 kubernaut service
ConfigMaps to how the kubernaut-operator handles it. Each field is categorized:

- **CR**: exposed in the `Kubernaut` CR spec; user-tunable with a default
- **HARDCODED**: pinned by the operator to a cluster-friendly value
- **DERIVED**: computed from other CR fields (namespace, BYO secrets, monitoring toggle)
- **BYO**: user-provided ConfigMap referenced by name; operator skips generation
- **REMOVE**: field deleted upstream in v1.4

All YAML keys use **camelCase** per ADR-030.

## 1. Kubernaut Agent

Upstream: `internal/kubernautagent/config/config.go`

### Breaking changes in v1.4

- Flat config (`server`, `investigator`, `dataStorage`, `tools`) replaced by
  3-domain layout: `runtime`, `ai`, `integrations`.
- `investigator` renamed to `investigation` under `ai`.
- SDK config (`sdk-config.yaml`) replaced by LLM runtime ConfigMap
  (`kubernaut-agent-llm-runtime` / `llm-runtime.yaml`).
- Default `maxTurns` raised from 15 to 40.
