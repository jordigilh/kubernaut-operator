## API and dependency versions

- **Kubernaut CR API:** `kubernaut.ai/v1alpha1`
- **Embedded CRD and types:** `github.com/jordigilh/kubernaut` module at **v1.4.0** (see `go.mod`)

## Default and example configuration

Authoritative field defaults are defined in the Kubernaut CRD for v1.4.0. A full illustrative instance, including bring-your-own PostgreSQL and Valkey secrets and policy ConfigMaps, is provided in `config/samples/v1alpha1_kubernaut.yaml`. Review that sample when building environment-specific baselines; demonstration values may enable features (for example monitoring or Routes) that differ from minimal defaults.

## Feature flags

| Feature | Default | CR field |
|---------|---------|----------|
| OCP Monitoring integration | `false` | `spec.monitoring.enabled` |
| NetworkPolicies | `false` | `spec.networkPolicies.enabled` |
| Ansible or AAP integration | `false` | `spec.ansible.apiURL` (presence enables integration) |
| Kubernaut Agent audit | `true` | `spec.kubernautAgent.audit.enabled` |
| Remediation dry-run | `false` | `spec.remediationOrchestrator.dryRun` |
| OAuth2 LLM authentication | `false` | `spec.kubernautAgent.llm.oauth2.enabled` |
| Alignment check | `false` | `spec.kubernautAgent.alignmentCheck.enabled` |
| Safety sanitization | `true` | `spec.kubernautAgent.safety.sanitization.*` |
| Safety anomaly detection | `true` | `spec.kubernautAgent.safety.anomaly.*` |
| Gateway Route | `false` | `spec.gateway.route.enabled` |

**OpenAPI default caveat:** The v1.4.0 CRD marks `spec.monitoring.enabled` and `spec.gateway.route.enabled` with OpenAPI default `true` when applying schema defaults. This baseline adopts **`false`** as the approved configuration for controlled environments. Hardened manifests under change control should set these fields explicitly rather than relying on admission defaulting.

## Breaking changes from v1.3

- **Kubernaut Agent configuration:** Flat configuration layout replaced by a three-domain structure (`runtime`, `ai`, `integrations`).
- **SDK configuration:** SDK-oriented settings are supplied through the LLM runtime ConfigMap pattern rather than legacy SDK-only ConfigMaps.
- **Default `maxTurns`:** Default maximum agent turns increased from fifteen to forty.

## Supply chain assurance

- **Image pinning:** Workload images are pinned by digest in `RELATED_IMAGE_*` variables as listed above.
- **Software bill of materials:** CycloneDX JSON SBOMs are attached to GitHub Releases for this repository.
- **Signing:** Container images are signed with Cosign in keyless mode using GitHub Actions OIDC.
- **Provenance:** Build provenance is attested using SLSA-oriented practices via `actions/attest-build-provenance` in GitHub Actions (SLSA levels one to two scope for generated attestations).
